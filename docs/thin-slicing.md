# Thin Slicing — MM3e Online (Holistic)

> Slicing strategy: **vertical by mechanic domain**, spine-first. Each increment delivers a
> playable or demonstrable capability end-to-end. Powers are sliced by effect mechanic (3 slices)
> rather than delivered all at once.

---

## Increment 1: Bare-Bones Playable Hero

**Outcome:** A GM can set a series *Power Level*, a player can build a stripped-down hero
(abilities + defenses only, one motivation), and the pair can run a basic trait check or
resistance check and resolve the outcome. Manual PP tracking is fine — one end-to-end code path.

**Slicing notes:** No skills, advantages, or powers. Complications are minimum (motivation
only). Check resolution covers basic trait checks and resistance checks only — opposed
checks, team checks, attack checks, and active defence checks are deferred to the combat
increment. PL caps are enforced from the start.

**Stories:**
- Set Series Power Level
- Derive Starting Power Points from Power Level
- Spend Power Points on Ability Ranks
- Spend Power Points on Defense Ranks
- Validate Power Points Total Balance
- Enforce Attack and Effect Cap
- Enforce Dodge and Toughness Cap
- Enforce Parry and Toughness Cap
- Enforce Fortitude and Will Cap
- Develop Hero Concept
- Choose Motivation Complication
- Validate Minimum Complication Requirement
- Approve Completed Hero for Play
- Set Ability Rank
- Derive Base Defense Rank from Ability
- Make Trait Check
- Grade Check Result by Degree
- Make Resistance Check Against Effect
- Roll Initiative Check
- Determine Turn Order from Initiative Results
- Take Standard Action on Turn
- Take Move Action on Turn

## Increment 2: Affliction, Damage, and Full Combat Loop

**Outcome:** Players can configure *Affliction* and *Damage* power effects, run a complete
combat round with conditions applied and recovered, and resolve dying/stabilization.
The full turn economy is exercised: initiative → action → attack → resistance check →
condition → recovery.

**Slicing notes:** Closes the combat loop and adds the two primary attack mechanics.
Defers area effects, team attacks, and hazards as optional depth.

**Stories:**

- Make Team Check with Combined Effort
- Make Active Defence Check
- Make Close Attack Check Against Parry
- Make Ranged Attack Check Against Dodge
- Resolve Attack Check Against Defense Class

- Resolve Affliction Resistance Check and Apply Condition Degree
- Recover from Affliction Condition at End of Turn
- Select Damage Effect for Close Combat
- Select Blast Effect for Ranged Energy Attack
- Resolve Damage Resistance Check and Apply Damage Condition
- Apply Condition to Character
- Supersede Condition in Chain
- Roll Resistance Check Against Ongoing Effect to Remove Conditions
- Remove Condition When Source Effect Ends
- Resolve Toughness Resistance Check Against Damage
- Apply Cumulative Toughness Penalty
- Apply Damage Condition by Degree of Failure
- Transition Incapacitated Target to Dying
- Apply Minion Defeat Rule on Any Resistance Failure
- Recover from Damage in Conflict
- Remove Damage Condition After Rest
- Resolve Ongoing Effect Resistance Check at End of Turn

- Take Reaction Outside Turn
- Determine Critical Hit When Total Meets Defense
- Apply Critical Hit Effect Choice
- Defend Against Incoming Attacks
- Roll Fortitude Check to Stabilize While Dying
- Stabilize Dying Ally with Treatment Check

## Increment 4: Attack Powers — Ranged, Weakening, Nullify

**Outcome:** Players can configure the remaining offensive power effects (*Blast*,
*Deflect*, *Weaken*, *Nullify*) with the full *Power Cost Formula* (extras and flaws),
and resolve their resistance checks in play.

**Slicing notes:** *Affliction* and *Damage* shipped in Increment 2. This slice adds
the rest of the offensive palette and introduces extras, flaws, and descriptors —
foundational mechanics for all subsequent power work.

**Stories:**
- Spend Power Points on Powers
- Select Deflect Effect for Ranged Protection
- Select Nullify Effect and Define Target Descriptor
- Select Weaken Effect and Define Target Trait
- Resolve Weaken Resistance Check and Reduce Target Trait
- Recover Weakened Trait at One Point per Round
- Resolve Nullify Opposed Check Against Target Effect Rank
- Apply Extra Modifier to Increase Per-Rank Cost
- Apply Flaw Modifier to Decrease Per-Rank Cost
- Prevent Per-Rank Cost Dropping Below Minimum via Flaw
- Apply Flat Cost Modifier to Power
- Apply Linked Effect Modifier to Power
- Assign Origin Descriptor to Power Set
- Assign Source Descriptor to Power Effect
- Assign Result Descriptor to Power Effect
- Match Power Descriptor Against Immunity Effect Coverage
- Match Power Descriptor Against Nullify Effect Target
- Approve Descriptor Interaction Between Powers

## Increment 5: Defense and Recovery Powers

**Outcome:** Players can configure *Protection*, *Immunity*, *Regeneration*, and
*Immortality* effects; the system resolves immunity checks and regeneration recovery
phases in play.

**Stories:**
- Select Protection Effect for Toughness Bonus
- Select Immunity Effect and Define Covered Effect Set
- Configure Partial Immunity for Half-Effect Protection
- Select Regeneration Effect for Progressive Damage Recovery
- Select Immortality Effect for Death Recovery
- Apply Immunity Check Against Matching Power Descriptor
- Remove Toughness Penalty via Regeneration Recovery Phase
- Remove Damage Condition via Regeneration Condition Phase
- Recover from Death via Immortality After Time Elapses

## Increment 6: Mobility, Sensory, and Power Arrays

**Outcome:** Players can configure movement powers (*Flight*, *Speed*, *Teleport*, etc.),
sensory powers, and organize powers into standard and dynamic arrays with proper
mutual-exclusivity and switching rules.

**Stories:**
- Select Flight Effect for Aerial Movement
- Select Speed Effect for Ground Velocity Enhancement
- Select Teleport Effect for Instantaneous Relocation
- Select Movement Effect and Choose Special Movement Mode
- Calculate Flight Speed Rank from Effect Rank
- Validate Teleport Destination as Known or Accurately Sensed
- Select Senses Effect and Allocate Sense Enhancements
- Select Concealment Effect and Define Affected Sense Type
- Build Power Array with Base Effect
- Add Alternate Effect to Array within Primary Effect Cost Limit
- Add Dynamic Alternate Effect for Simultaneous Use
- Switch Active Effect in Array as Free Action
- Enforce Array Mutual Exclusivity Rule for Non-Dynamic Effects
- Propagate Array Disable Equally to All Alternate Effects
- Reallocate Power Points Among Dynamic Array Effects per Turn
- Enforce Dynamic Effect Point Total Against Array Pool Size

**Outcome:** Players can allocate *Equipment Points*, acquire weapons, armor, and gear,
build *Devices* (removable powers with cost discounts), invent temporary devices,
